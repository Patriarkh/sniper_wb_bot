SniperBot — это Telegram-бот для анализа товаров на Wildberries с использованием API MPStats. Бот предназначен для автоматизации поиска товаров, анализа данных и формирования базы для пользователей.
_________________________________________

Основной функционал
1. Регистрация API-ключа MPStats:

Пользователи регистрируют свои персональные ключи API MPStats.
Ключи сохраняются в базе данных для последующего использования.

2. Формирование базы данных товаров:

Пользователь задает параметры фильтрации:
Количество товаров.
Дата первого отзыва.
Диапазон выручки.
Бот формирует запрос к MPStats, анализирует данные и сохраняет результаты в базе.

3. Ежедневная проверка новых товаров:

Бот автоматически проверяет наличие новых товаров, соответствующих заданным фильтрам.
Результаты проверки отправляются пользователям в виде сообщений с описанием товара и фотографиями.

4. Управление фильтрами:

Пользователи могут удалять текущие фильтры и данные через команду /delete.

5. Рассылка сообщений:

Администратор может отправлять массовые сообщения всем пользователям.

6. Проверка подписки на канал:

Для использования бота пользователь должен быть подписан на указанный канал.
_________________________________________

Обзор файлов

1. sniper.py — основной файл
Основная точка входа в приложение.
Управляет диалогами, командными обработчиками и расписанием задач.
Реализует:
Регистрацию пользователей.
Команду /delete для удаления фильтров.
Команду /broadcast для рассылки сообщений.
Ежедневную проверку базы данных и уведомления.

2. reg_api_mpstat.py
Управляет регистрацией API-ключей.
Включает функции для сохранения и проверки ключей.
Диалог для регистрации ключа активируется командой /start.

3. formirovanie_zaprosa.py
Реализует диалог для задания фильтров товаров.
Обрабатывает:
Количество товаров.
Дату первого отзыва.
Диапазон выручки.
После задания параметров отправляет запрос к MPStats.

4. handle_request.py
Запускает обработку запросов к MPStats.
Используется для генерации задач в JobQueue.

5. common.py
Содержит вспомогательные функции, такие как:
Запросы к API MPStats.
Логирование.
Работа с базой данных.

6. database_utils.py
Управляет взаимодействием с базой данных SQLite:
Создание таблиц.
Сохранение и удаление данных.
Получение ключей API для пользователей.

7. every_day.py
Реализует логику для ежедневной проверки новых товаров.
Сравнивает новые данные с текущей базой и отправляет результаты пользователям.

8. helpers.py
Вспомогательные функции:
Отправка длинных сообщений.
Логирование действий.

9. get_member.py
Проверяет подписку пользователя на канал.
Реализует декоратор @subscription_required для ограничения доступа.

Логика работы
1. Регистрация:

Пользователь запускает бота с помощью команды /start.
Вводит свой API-ключ MPStats.
Ключ сохраняется в базе.

2. Формирование запроса:

Пользователь вводит параметры фильтрации через команду /start_bot.
Бот сохраняет параметры в базе и отправляет запрос к MPStats.

3. Получение данных:

Результаты запроса сохраняются в базу данных.
Пользователь получает описание товара, выручку, дату первого отзыва, ссылку на товар и изображение.

4. Ежедневная проверка:

Запускается автоматически в установленное время.
Проверяет наличие новых товаров, соответствующих фильтрам.

5. Управление фильтрами:

Пользователь может удалить фильтры и данные через команду /delete.

6. Рассылка:

Администратор отправляет массовые сообщения через /broadcast.
